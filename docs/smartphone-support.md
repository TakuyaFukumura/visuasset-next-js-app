# スマホ対応仕様

## 概要

本資料は visuasset-next-js-app におけるスマートフォン対応の実装仕様をまとめたものです。

---

## 1. ハンバーガーメニュー

### 目的

スマートフォンなどの小画面デバイスでは、ヘッダーのナビゲーションリンクを横並びに表示するとレイアウトが崩れる。
ハンバーガーメニューを導入することで、小画面でも快適にナビゲーションできるようにする。

### 仕様

| 項目 | 内容 |
|------|------|
| 表示条件 | 画面幅 640px 未満（Tailwind CSS の `sm` ブレークポイント未満） |
| メニューアイコン | ☰（閉じた状態） / ✕（開いた状態） |
| メニュー展開方式 | ヘッダー直下にドロップダウン表示 |
| メニュー閉じる条件 | ハンバーガーアイコンを再クリック、またはリンクをクリックしたとき |

### デスクトップ表示（640px 以上）

- ナビゲーションリンクをヘッダー内に横並びで表示（現行通り）
- ハンバーガーボタンは非表示

### モバイル表示（640px 未満）

- ヘッダー内のナビゲーションリンクを非表示
- ハンバーガーボタンをヘッダーに表示
- ハンバーガーボタンを押すとナビゲーションリンクがドロップダウン表示される
- ドロップダウン内でリンクをクリックすると自動的にメニューが閉じる

### 対象ファイル

- `src/app/components/Header.tsx`

---

## 2. モバイルでの余白最小化

### 目的

スマートフォンでの表示において、ページの余白が大きく、グラフ描画エリアが狭くなっている。
余白を必要最小限に抑えてグラフを見やすくする。

### 仕様

ページ全体のパディングをレスポンシブ対応させる。

| ブレークポイント | パディング |
|----------------|-----------|
| モバイル（デフォルト） | `p-2`（0.5rem） |
| `sm`（640px 以上） | `p-6`（1.5rem） |

### 対象ファイル

- `src/app/page.tsx`
- `src/app/portfolio/page.tsx`
- `src/app/simulation/page.tsx`

---

## 3. 実装方針

- Tailwind CSS のレスポンシブプレフィックス（`sm:` など）を使用してスタイルを切り替える
- JavaScript によるリサイズ検知は行わず、CSS のみで対応する（ハンバーガーメニューの開閉のみ React の `useState` を使用）
- 既存のデスクトップ表示に影響を与えないよう、モバイルファーストのアプローチで実装する
